Создай новый экран (view) в проекте UzMashLizing по следующему плану.

Название экрана: $ARGUMENTS

## Шаги

1. **Создай папку и компонент**
   - Папка: `src/views/<kebab-case-name>/`
   - Основной файл: `src/views/<kebab-case-name>/<PascalCaseName>.tsx`
   - Структура компонента — функциональный, TypeScript, пропсы через interface

2. **Добавь маршрут в `src/configs/routes.config/routes.config.ts`**
   - Импортируй с `lazy(() => import(...))`
   - Укажи `authority: []` — я заполню роли сам, но спроси у меня какие роли нужны
   - Пример структуры:
   ```typescript
   {
     key: '<snake_case_key>',
     path: '/<kebab-path>',
     component: lazy(() => import('@/views/<name>/<Component>')),
     authority: [/* роли */]
   }
   ```

3. **Добавь пункт в навигацию `src/configs/navigation.config/index.ts`**
   - Используй `NAV_ITEM_TYPE_ITEM` для простого пункта
   - Используй `NAV_ITEM_TYPE_COLLAPSE` если нужно подменю
   - Укажи иконку из доступных в проекте (HeroIcon)
   - Спроси у меня: какая иконка, какой заголовок, в какой раздел навигации

## Перед созданием задай мне вопросы:
- Какие роли должны иметь доступ?
- Это самостоятельный раздел или подраздел существующего?
- Какую иконку использовать?
- Нужен ли мок-эндпоинт для этого экрана?

## Правила
- Нет захардкоженных строк — только через `translateKey`
- Данные через props, не через прямые вызовы store внутри компонента
- TypeScript строгий: нет `any`, все типы явные
