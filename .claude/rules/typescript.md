# TypeScript — правила проекта

Строгий режим (`strict: true`). Запрет `any` — всегда указывай явные типы.

## Типы данных

- **Денежные суммы** — хранить как `string`, никогда `number`. Поля: `price`, `total_amount`, `deposit_percentage`, `rent_percent` и аналогичные.
- **Даты** — принимают `Date | string`. Использовать единый форматтер `dateUtils.ts` во всём проекте.
- **Файлы** — использовать везде тип `FileType`. Не создавать отдельных типов для файлов.

## Двуязычность

Обязательно заполнять оба поля: `name_ru` и `name_uz`. Это касается всех сущностей с именами (регионы, типы техники, производители и т.д.).

## Компоненты

Делай компоненты независимыми от источника данных — передавай данные через props, не обращайся к стору напрямую внутри переиспользуемых компонентов.
