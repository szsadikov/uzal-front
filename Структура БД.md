# Структура БД — UzMashLizing

> Документ описывает все сущности, их поля и типы, используемые в проекте.
> При создании дашбордов и мок-данных опираться **только** на поля из этого документа.
> Источники: `src/@types/`, `src/mock/fakeApi/`, `src/services/`

---

## Важные правила

- Все денежные суммы хранятся как **`string`** — никогда не использовать `number` для денег
- Даты принимают **`Date | string`** — использовать единый форматтер
- **`FileType`** используется везде для файлов — не создавать отдельных типов
- Двуязычность обязательна — всегда заполнять **`name_ru`** и **`name_uz`**

## Связи между сущностями

```
Contract ──────────── Tech (tech_obj)
Contract ──────────── ContractApplication (contract_application)
Contract ──────────── Lessee (через client_*)
ContractApplication ── Branch
ContractApplication ── Tech
ContractApplication ── Lessee
ContractApplication ── Vote[]
Lessee ─────────────── User (profile)
Lessee ─────────────── Region
Branch ─────────────── Region
Branch ─────────────── City
User ────────────────── Region
User ────────────────── Role
PaymentOverdueNotice ── Contract (по id)
PaymentOverdueNotice ── Branch
BegunokDetail ────────── Contract (по id)
TechMonitorTaskDetail ── TechMonitoring (monitoring)
```
